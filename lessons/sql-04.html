Order By

Pelajaran 4: Mengurutkan Hasil Query

üìö Order By - Pelajaran 4

Apa itu ORDER BY?

ORDER BY digunakan untuk mengurutkan hasil query berdasarkan satu atau lebih kolom. Hasil dapat diurutkan secara ascending (naik) atau descending (turun).

Syntax ORDER BY

SELECT kolom1, kolom2, ...
FROM nama_tabel
ORDER BY kolom1, kolom2, ... ASC|DESC;

ORDER BY Ascending (Urutan Naik)

Ascending (ASC) adalah urutan default. Data diurutkan dari nilai kecil ke besar, atau dari A ke Z untuk string.

-- Urutkan siswa berdasarkan nama (A-Z)
SELECT * FROM siswa ORDER BY nama;

-- Sama dengan:
SELECT * FROM siswa ORDER BY nama ASC;

-- Output:
-- | nama | umur | kota |
-- | Ani | 19 | Bandung |
-- | Budi | 20 | Jakarta |
-- | Citra | 21 | Surabaya |

ORDER BY Descending (Urutan Turun)

Descending (DESC) mengurutkan data dari nilai besar ke kecil, atau dari Z ke A untuk string.

-- Urutkan siswa berdasarkan umur (besar ke kecil)
SELECT * FROM siswa ORDER BY umur DESC;

-- Output:
-- | nama | umur | kota |
-- | Citra | 21 | Surabaya |
-- | Budi | 20 | Jakarta |
-- | Ani | 19 | Bandung |

ORDER BY Multiple Kolom

Anda dapat mengurutkan berdasarkan lebih dari satu kolom. Kolom pertama adalah prioritas utama.

-- Urutkan berdasarkan kota (ASC), kemudian berdasarkan umur (DESC)
SELECT * FROM siswa
ORDER BY kota ASC, umur DESC;

-- Output:
-- | nama | umur | kota |
-- | Ani | 19 | Bandung |
-- | Budi | 20 | Jakarta |
-- | Citra | 21 | Surabaya |

ORDER BY dengan WHERE

ORDER BY dapat dikombinasikan dengan WHERE untuk memfilter dan mengurutkan data.

-- Siswa dari Jakarta, diurutkan berdasarkan umur
SELECT * FROM siswa
WHERE kota = 'Jakarta'
ORDER BY umur DESC;

-- Siswa dengan umur > 19, diurutkan berdasarkan nama
SELECT nama, umur FROM siswa
WHERE umur > 19
ORDER BY nama ASC;

ORDER BY dengan Fungsi Agregat

ORDER BY dapat digunakan dengan GROUP BY untuk mengurutkan hasil agregat.

-- Hitung siswa per kota, urutkan dari jumlah terbanyak
SELECT kota, COUNT(*) as jumlah_siswa
FROM siswa
GROUP BY kota
ORDER BY jumlah_siswa DESC;

-- Output:
-- | kota | jumlah_siswa |
-- | Jakarta | 2 |
-- | Bandung | 1 |
-- | Surabaya | 1 |

ORDER BY dengan Column Number

Anda dapat mengurutkan menggunakan nomor kolom (posisi) bukan nama kolom.

-- Urutkan berdasarkan kolom ke-1 (nama)
SELECT nama, umur FROM siswa ORDER BY 1;

-- Urutkan berdasarkan kolom ke-2 (umur) descending
SELECT nama, umur FROM siswa ORDER BY 2 DESC;

LIMIT

LIMIT digunakan untuk membatasi jumlah baris yang dikembalikan. Sering dikombinasikan dengan ORDER BY.

-- 3 siswa dengan umur tertinggi
SELECT * FROM siswa
ORDER BY umur DESC
LIMIT 3;

-- Siswa pertama dalam urutan nama
SELECT * FROM siswa
ORDER BY nama
LIMIT 1;

-- Ambil 2 baris, mulai dari baris ke-2
SELECT * FROM siswa
ORDER BY nama
LIMIT 2 OFFSET 1;

ORDER BY NULL

Ketika mengurutkan, NULL biasanya muncul di awal atau akhir tergantung database.

-- Urutkan dengan NULL di akhir (BigQuery)
SELECT * FROM siswa
ORDER BY kota IS NULL, kota ASC;

Contoh Query Lengkap

Berikut adalah contoh query yang menggabungkan SELECT, FROM, WHERE, GROUP BY, HAVING, dan ORDER BY:

-- Kota dengan minimal 2 siswa, urutkan dari rata-rata umur tertinggi
SELECT
    kota,
    COUNT(*) as jumlah_siswa,
    AVG(umur) as rata_rata_umur
FROM siswa
GROUP BY kota
HAVING COUNT(*) >= 2
ORDER BY rata_rata_umur DESC;

Urutan Eksekusi Query SQL

Penting untuk memahami urutan eksekusi query SQL:

  1. FROM - Tentukan tabel
  2. WHERE - Filter baris
  3. GROUP BY - Kelompokkan baris
  4. HAVING - Filter grup
  5. SELECT - Pilih kolom
  6. ORDER BY - Urutkan hasil
  7. LIMIT - Batasi jumlah baris
üí° Tips:

ORDER BY ASC adalah default, jadi Anda tidak perlu menulisnya. Untuk performa terbaik, hindari mengurutkan berdasarkan kolom yang tidak di-index. Gunakan LIMIT untuk membatasi hasil jika dataset besar.

Ringkasan Pelajaran

  • ORDER BY mengurutkan hasil query
  • ASC (ascending) adalah default, DESC untuk turun
  • ORDER BY dapat menggunakan multiple kolom
  • Dapat dikombinasikan dengan WHERE dan GROUP BY
  • LIMIT membatasi jumlah baris yang dikembalikan
  • Urutan eksekusi query penting untuk dipahami
  • ORDER BY dengan LIMIT sangat berguna untuk mendapat data top-N
